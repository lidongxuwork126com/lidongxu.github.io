# 函数名就是变量名

讲回调函数之前, 先理解这个概念, 其实我们声明的function的名字, 就是一个普通变量

function eat () {};             // eat 是函数名, 也是一个变量
var eat = function () {};       // eat 是变量, 接收了一个函数体
// 都不耽误调用 eat();
1
2
3

那么我们就可以把一个函数体赋予给一个形参了

function say(eat)){
    eat();
};
say(function(){
    console.log("1行代码触发了我");
});
// 调用say方法时相当于
// eat变量 = function(){console.log("1行代码触发了我");}
// 所以第二行, eat(), 相当于在调用第5行代码执行

所以, 当say()调用时, 传进去函数体, 然后在eat()调用时叫做回调

总结: 当函数内调用形参方法执行时, 这个过程就叫回调

# 注意

  1. 当回调函数嵌套使用过多, 造成代码难以维护, 所以不要过深的使用回调函数
  2. 回调函数在Ajax和定时器计时器以及nodeJS和异步操作中使用非常广泛
