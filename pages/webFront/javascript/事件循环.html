# Javscript事件循环

首先, Javscript是单线程, 非阻塞的.
单线程: 防止多线程同时操作同一个dom, 造成不一致性
非阻塞: 把异步操作, 加入到任务队列(回调队列)中, 在执行栈空闲时, 取出队列异步函数调用, 往返的过程叫事件循环.

web apis -> 异步事件
任务队列 -> 内存中组织代码的一种数据格式

# 异步任务

异步任务队列分

  • 微观任务
    • Promise的任务
  • 宏观任务
    • setTimeout()
    • setInterval()
    • XMLHttpRequest()

看如下例子打印顺序 主线程+微观任务Promise+宏观任务setTimeout/ajax

        console.log(222);
        async function one(){
            console.log(111);
            await two(); 
            console.log(888);
        }
        async function two(){
            console.log(999);
        }
        setTimeout(()=>{
            console.log(1010);
        }, 0);
        console.log(333);
        one();
        let p = new Promise((resolve, reject) => {
            console.log(444);
            resolve();
            setTimeout(()=>{
                console.log(11);
            }, 0);
            console.log(666); 
        })
        p.then(res => {
            console.log(555);
        })
        console.log(777);
        // 执行顺序: 主线程 -> 微观任务 -> 宏观任务 (从上往下)
        // 遇到setTimeout等宏观任务插入到队列尾部, 如果是resolve/reject的Promise微观任务按先后顺序放在队列头部

        // 详细讲解打印顺序
        // []
        // 遇到第1行, 222打印
        // 遇到第10行, setTimeout宏观任务, 先装在任务队列中, 等执行栈主线程跑完所有代码再回来执行任务队列
        // [10行setTimeout]
        // 遇到第13行, 333打印
        // 遇到第14行, 调用one方法执行, 111打印
        // 遇到第24行, 调用two方法执行, 999打印, async函数总是返回Promise所以,await等待的代码, 进入到任务队列, 而且微任务总是在宏观任务之前(前提:微任务和宏观任务没互相嵌套)
        // [4行await, 10行setTimeout]
        // 遇到第15行, new Promise()里回调函数马上执行, 444打印
        // resolve装到任务队列中
        // [4行await, 17行resolve, 10行setTimeout]
        // 遇到第19行, setTimeout宏观任务, 在第一层队列后
        // [4行await, 17行resolve, 10行setTimeout, 19行setTimeout]
        // 遇到21行, 666打印
        // 遇到26行, 777打印
        // 主线程执行栈, 所有代码执行完毕
        // 222 333 111 999 444 666 777
        
        // 开始执行任务队列中代码
        // 回到4行await, 放行 888打印
        // 回到17行resolve, 555打印
        // 回到10行setTimeout, 0秒马上触发, 1010打印
        // 回到19行setTimeout, 0秒马上触发, 11打印
        
        // 执行顺序 
        // 在宏观任务队列中, 还是要按照时间顺序来执行的
        // 事件循环中, 微任务清空以后, 才会执行宏任务, 而且他们可以互相嵌套
